## 项目概述

**CTI-RAG** 是一个面向网络威胁情报（CTI）问答的图谱增强系统，核心目标是解决传统 RAG 在多跳关系问答中的召回不完整、上下文冗余和可解释性不足问题。  
系统采用“向量检索 + 知识图谱召回 + 强化学习剪枝 + 大模型生成”的端到端链路，支持图、向量双检索协同。

![](images/projects/CTI-rag.png)

## 系统架构

在检索层，先通过向量检索定位候选语义片段，再基于知识图谱补全实体关系与多跳路径，形成更具结构化的信息上下文。  
在生成层，统一接入 LLM Gateway，实现多模型路由、超时控制、熔断与降级，提升模型服务抖动场景下的可用性。

![](images/projects/CTI-db.png)

## 关键实现

1. 设计并实现面向 CTI 问答的 GraphRAG 流程，增强多跳关系问题的答案完整性与可解释性。  
2. 基于 RabbitMQ 实现向量化、子图构建、批量评测等异步任务处理，通过任务拆分、重试退避、死信队列优化高并发阻塞与超时。  
3. 使用 Redis 进行检索结果缓存、会话状态管理与接口限流，通过分布式锁、TTL 分级和本地缓存兜底降低热点请求下的重复计算与缓存击穿。  
4. 引入强化学习边级剪枝策略，在保证回答效果的前提下，显著降低子图规模与推理 Token 开销。

![](images/projects/CTI-Kdb.png)

## 效果指标

- 平均子图规模从 **14.12** 降至 **6.50**，下降 **53.96%**  
- 平均 Token 消耗从 **4936.24** 降至 **2564.71**，下降 **48.04%**

上述结果表明：强化学习剪枝策略能够在不明显影响答案效果的情况下，显著提升系统推理效率与工程可用性。

## 总结

CTI-RAG 的核心价值在于把“结构化图知识”和“语义向量检索”融合到同一问答闭环中，并通过异步任务体系与网关治理能力保证工程落地。  
这使系统不仅能回答“是什么”，还能更清晰地回答“为什么”与“如何关联”。
